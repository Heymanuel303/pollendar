# Phase 1: Vote|Results toggle + Vote-tab refactor + closed-poll rework

**Plan:** [2026-06-18-participant-matrix](00-overview.md)
**Depends on:** none
**Execution:** solo

## Context
The public poll (`/p/:publicToken`) currently shows a single tri-state voting form, and when a poll is closed it hard-replaces that form with a "This poll is closed" notice. The feature goal is a **Vote | Results** view on the public poll: keep tri-state voting, and add a per-participant matrix showing who voted and what they picked, visible to anyone with the share link for **open and closed** polls. This phase is the local-state scaffolding only — it adds the segmented Vote|Results toggle, moves the existing form into the **Vote** tab, and makes a closed poll *disable* Vote inputs instead of hiding the surface. The actual per-participant matrix (`ParticipantMatrix.vue`) and its data come from later phases; that data is supplied by the **who-voted-endpoint** plan (a `publicPollStore.loadParticipants(token)` action backing `GET /api/public/polls/:token/participants-responses`, returning `displayName` + per-slot answers, **never email**). Phase 1 must ship without any of that.

## Objective
Add a Vote|Results segmented toggle (persisted in localStorage), refactor the existing voting form into the "Vote" tab (PollSlotRow + the voter's own editable row), and change the closed-poll branch to disable Vote inputs while keeping the surface visible. Ships with NO backend dependency.

## Files to touch
- `frontend/src/views/PublicPoll.vue` — add the Vote|Results segmented toggle below the poll intro; wrap the existing availability/about-you form in a `view === 'vote'` panel; rework the closed-poll branch (lines 146-155) so it no longer replaces the form but disables Vote inputs and shows a "closed" banner; add a placeholder **Results** panel (`view === 'results'`) with the existing live-results data (bloom/leaning) and a "Per-person results coming soon" stub where `ParticipantMatrix` will mount in phase 2; thread an `isOpen`-derived `disabled` flag down to `PollSlotRow`.
- `frontend/src/lib/viewMode.ts` — **new** tiny localStorage helper mirroring `participantToken.ts` (try/catch on every access). Persists the Vote|Results choice per poll under key `pollendar:vm:<pollPublicToken>`. Exports `getViewMode(pollPublicToken): ViewMode | null` and `saveViewMode(pollPublicToken, mode): void`, with `type ViewMode = 'vote' | 'results'`.
- `frontend/src/components/PollSlotRow.vue` — add an optional `disabled?: boolean` prop; when true, render the `AvailabilityToggle` inside a `<fieldset :disabled="disabled">` (or pass a new `disabled` prop down to `AvailabilityToggle` — see Steps) and apply a muted/`opacity` treatment so a closed poll's Vote tab is visibly read-only.
- `frontend/src/components/AvailabilityToggle.vue` — add an optional `disabled?: boolean` prop (currently has none); when true, set `:disabled="disabled"` on each `<button>`, guard `select()` to no-op, and add `aria-disabled`/muted styling. (Only if the fieldset approach in `PollSlotRow` is not chosen — keep one mechanism, not both.)

## Steps
1. **Create `frontend/src/lib/viewMode.ts`.** Copy the structure of `frontend/src/lib/participantToken.ts` exactly: a `const KEY_PREFIX = 'pollendar:vm:'`, a `keyFor(pollPublicToken)` helper, and try/catch-wrapped `getViewMode` / `saveViewMode`. `getViewMode` reads `localStorage.getItem(keyFor(token))`, returns it only if it is `'vote'` or `'results'`, else `null` (treat read failure or any unknown value as "no stored mode"). `saveViewMode` writes the mode, silently no-ops on quota/disabled storage. Export `type ViewMode = 'vote' | 'results'`. Per-poll keying (not a global `pollendar:view-mode`) is the decided contract — mirror `participantToken.ts`'s per-poll namespacing.
2. **Add the toggle state to `PublicPoll.vue` script.** Import `{ getViewMode, saveViewMode, type ViewMode } from '@/lib/viewMode'` and `{ watch }` from `vue` (already importing `computed, onMounted, reactive`). Add `const view = ref<ViewMode>('vote')`. In `onMounted` (after `store.load(token.value)` resolves and `token.value` is known), seed `view.value = getViewMode(token.value) ?? 'vote'`. Add a `watch(view, (mode) => saveViewMode(token.value, mode))` so every toggle persists. Add `function setView(mode: ViewMode) { view.value = mode }` for the buttons. **Do NOT import or call any `loadParticipants` action — it does not exist yet and is out of scope for this phase.**
3. **Add the segmented toggle markup** to `PublicPoll.vue`, between the "Poll intro" block (ends ~line 144) and the form. Mirror the segmented-toggle pattern from `PollSlotRow`/`AvailabilityToggle`: an `inline-flex rounded-xl border border-line bg-canvas p-1` container with `role="group"` + `aria-label="View"`, containing two real `<button type="button">` children — "Vote" and "Results". Each button uses `aria-pressed="view === 'vote'"` (resp. `'results'`), active state `bg-pollen text-canvas shadow-glow` (or the existing `bg-yes`/active classes used in the tri-state group), inactive `text-dim hover:text-moonlight`, `@click="setView('vote')"` / `setView('results')`. The toggle renders for **both** open and closed polls (it is the only navigation between voting and the matrix surface).
4. **Wrap the existing Vote form in a `v-show="view === 'vote'"` panel.** The current `<template v-else>` (lines 157-220) holds the "Your availability" section, "About you" section, and the helper line. Keep that markup but make it the **Vote** panel and render it for closed polls too (so the surface stays visible). Use `v-show` (not `v-if`) so toggling tabs does not discard the in-progress `answers`/`form` state. Remove the `v-else` gating on poll status — status now controls *disabled*, not *presence*.
5. **Rework the closed-poll branch (current lines 146-155).** Delete the `<section v-if="!isOpen">…This poll is closed…</section>` card that replaces the form. Replace it with a slim **inline banner** rendered `v-if="!isOpen"` *inside* the Vote panel, above the availability section — e.g. a `rounded-xl border border-line bg-surface2 px-4 py-3 text-sm text-dim` strip reading "This poll is closed — voting is no longer open, but you can still view results." The availability section and About-you stay rendered but become non-interactive (Step 6).
6. **Disable the Vote inputs when closed.** Pass `:disabled="!isOpen"` to each `<PollSlotRow>` in the `v-for` (around current lines 172-181). In `PollSlotRow.vue` add `disabled?: boolean` to `defineProps`, and forward it: wrap the `<AvailabilityToggle>` in `<fieldset :disabled="disabled" class="contents">` **or** add a `disabled` prop to `AvailabilityToggle` — pick the `AvailabilityToggle` prop approach (cleaner styling control). In `AvailabilityToggle.vue` add `disabled?: boolean` to props, set `:disabled="disabled"` on every `<button>`, early-return from `select()` when `props.disabled`, and add a muted class (e.g. `opacity-60 cursor-not-allowed` on the container when disabled). Also disable the About-you `<input>`s for `displayName`/`email` with `:disabled="!isOpen"`. The sticky submit bar already only renders `v-if="poll && isOpen"` — leave that guard; a closed poll shows no submit affordance.
7. **Add the placeholder Results panel** with `v-show="view === 'results'"`, a sibling of the Vote panel. Reuse the already-available `results` ref + `bestSlotId`/`leaningLabel` computeds and `formatDate` to render a minimal best-slot/leaning summary (or simply re-state "Leaning so far"), then a clearly-labeled stub block: `rounded-2xl border border-dashed border-line bg-surface p-6 text-center text-dim` reading "Per-person results coming soon" with a comment `<!-- ParticipantMatrix mounts here in phase 2 -->`. **Do not** import `ParticipantMatrix.vue` (it does not exist yet) and **do not** call any participant-rows action. The Results panel must render for both open and closed polls.
8. **Self-check the no-backend-dependency invariant.** Grep the diff for `loadParticipants`, `participants`, `participants-responses`, `ParticipantMatrix` — none should be imported, referenced, or called in this phase. The only new module is `viewMode.ts`. All new state is local component/Pinia-free reactive state plus the localStorage view-mode helper.

## Verification
- `cd frontend && npm run build && npm run lint`
- Manual (open poll): load `/p/:token`, confirm Vote tab shows the availability form and submit bar; switch to Results, confirm the placeholder + leaning summary; reload the page and confirm the last-selected tab is restored from localStorage (`pollendar:vm:<token>`).
- Manual (closed poll): load a completed/cancelled poll's `/p/:token`; confirm the Vote tab is still visible with a "poll closed" banner, the tri-state toggles and name/email inputs are disabled (not clickable), no submit bar shows, and the Results tab still renders.
- Manual at phone width (~375px): confirm both tabs and the toggle are usable and not clipped.
