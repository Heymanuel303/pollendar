# Phase 1: Pure calendar day-picker + calendar-left / editable-list-right layout

**Plan:** [poll-editor-calendar-list](00-overview.md)
**Depends on:** none
**Execution:** solo

## Context
The poll editor (`/polls/new` and `/polls/:id/edit`) currently shows a `Calendar | List` toggle that swaps two editors over one shared `dates` ref, plus a static preview sidebar (and a phone bottom-sheet) that holds the primary action. This phase redesigns that screen toward the feature goal: the candidate-times section becomes a **pure calendar day-picker** (no bulk-apply panel, no toggle) on the left with the Create/Save action beneath it, and the always-editable `DateSlotEditor` list moves to the right as the surface where users add custom slots and labels. The preview is removed entirely. The shared `dates` (`PollDateInput[]`) ref remains the single source of truth bound by both editors, so `CreatePollPayload`/`UpdatePollPayload` stay byte-identical and edit-mode vote-lock guards are preserved.

## Objective
Strip `CalendarDateEditor` down to a tap-to-toggle day-picker (tapped days get a fixed default slot) and restructure `PollEditor` into a calendar-left (with anchored Create/Save action) / editable-list-right layout — removing the toggle, the preview sidebar, and the phone preview bottom-sheet — then update the two spec files to match.

## Files to touch
- `frontend/src/components/CalendarDateEditor.vue` — remove the `SlotPresetChips` import + bulk-apply panel markup; keep `toggleDate()`, `defaultSlots()`, `activeSlots` (now serving only the fixed default slot); remove `applyToSelected()` and `selectedCount`'s only-for-button uses if any go dead; preserve edit-mode vote-lock in `toggleDate()`.
- `frontend/src/views/PollEditor.vue` — remove the `Calendar | List` toggle, the desktop `<aside>` preview, the phone "Show preview" button + Teleport bottom-sheet; delete the now-dead `editorView` ref + `watch`, `getEditorView`/`saveEditorView`/`EditorView` import, `showPreview` ref, `previewRows` computed, `closesPreview` computed; flip the grid to left=calendar+action / right=list; anchor the action beneath the calendar.
