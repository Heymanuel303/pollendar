# Phase 1: Scaffold & design system (app shell, Tailwind v4 tokens, API client, UI primitives)

**Plan:** [frontend-build-out](00-overview.md)
**Depends on:** none
**Execution:** solo

## Context
Pollendar finds the time everyone can make. This plan (PLAN.md Phase 7) builds the real Vue 3.5 + Vite + TypeScript SPA that consumes the finished NestJS backend at base path `/api` with cookie credentials. `frontend/` currently holds only a `README.md`, so this first phase must scaffold the project from scratch and lay the foundation every later view/component depends on: the "Dusk Calendar" design system (DESIGN-UI.md) wired the Tailwind v4 **CSS-first** way вАФ `@import "tailwindcss"` + an `@theme {}` token block, **no `tailwind.config.js`, no PostCSS, no v3 `@tailwind` directives** вАФ plus self-hosted fonts, the motif helper classes, the app shell + top nav, a credentials- and BigInt-safe fetch client, and the base UI primitives. Later phases (auth flow, dashboard, poll editor, public poll, results) compose what this phase establishes.

## Objective
Scaffold `frontend/` (Vue 3.5 + Vite + TS strict + Pinia 3 + vue-router 4), wire the Dusk Calendar Tailwind v4 `@theme` tokens + motif classes + self-hosted Space Grotesk/Inter fonts, and ship the app shell with top nav, a credentials+BigInt-safe API client, shared API/util types, and the base UI primitives (Button, Input, Field, Card, Pill).

## Steps
1. **Scaffold.** From `frontend/`'s parent, run `npm create vue@latest frontend` choosing TypeScript, Router, Pinia, ESLint (+ Prettier); preserve the existing `README.md`. `cd frontend && npm install`. Confirm Vue is 3.5.x, Pinia 3.x, vue-router 4.x, TS strict (`tsconfig.app.json` extends `@vue/tsconfig` with `"strict": true`) вАФ pin exact versions from the generated lockfile.
2. **Install design deps.** `npm install tailwindcss @tailwindcss/vite @fontsource/space-grotesk @fontsource/inter`.
3. **Wire Tailwind v4 (CSS-first).** Edit `vite.config.ts` to add `tailwindcss()` to `plugins` (after `vue()`). Do **not** create `tailwind.config.js` or a PostCSS config. Add `server: { port: 5173, proxy: { "/api": { target: "http://localhost:3000", changeOrigin: true } } }` so dev fetches to `/api` reach the backend with cookies treated as same-origin.
4. **Author `src/assets/main.css`.** First line `@import "tailwindcss";` then the `@theme {}` block ported verbatim from DESIGN-UI.md "Implementing in the real app" (`--color-canvas:#14122B`, `--color-surface:#211E40`, `--color-surface2:#2B2752`, `--color-line:#36325E`, `--color-pollen:#FFC857`, `--color-pollen-deep:#F0A93B`, `--color-moonlight:#F4F2FF`, `--color-dim:#B8B3DE`, `--color-mute:#7E79AE`, `--color-yes:#FFC857`, `--color-maybe:#B8B3DE`, `--color-no:#3A3563`, `--color-mint:#6FE0B0`, `--color-coral:#FF7A6B`, `--font-display:"Space Grotesk",вА¶`, `--font-sans:"Inter",вА¶`, `--shadow-glow:0 0 0 1px rgb(255 200 87 / .55), 0 0 28px rgb(255 200 87 / .32)`). Add `--shadow-card:0 8px 24px rgb(0 0 0 / .35)`. Set base `body { font-family: var(--font-sans); }`.
5. **Add motif classes** below `@theme` in `src/assets/main.css` (`@layer components` / `@utility`): `.bg-dusk` = `canvas` base + radial pollen-dot grain + faint purple gradients; `.pollen-dot` = round gold radial-gradient marker with glow (default size set by callers, e.g. `h-2.5 w-2.5` header, `h-3 w-3` grid); `.bloom` = `box-shadow: var(--shadow-glow)`; `.bloom-bg` = faint radial pollen wash; `.num` = `font-family: var(--font-display)` for numerals inside body text. Define `@keyframes settle` (upward drift + fade) and `@keyframes bloom` (glow grows in), 150вАУ250ms `ease-out`, all wrapped in `@media (prefers-reduced-motion: reduce)` to disable and show final state.
6. **`src/main.ts`.** Import `@fontsource/space-grotesk` (400/500/600/700) and `@fontsource/inter` (400/500/600), then `import "./assets/main.css"`, `createApp(App)`, `app.use(createPinia())`, `app.use(router)`, `app.mount("#app")`. Ensure `index.html` `<title>` is "Pollendar" and the favicon is the brand pollen dot.
7. **`App.vue` shell.** Root `<div class="bg-dusk min-h-screen text-moonlight">` containing `<AppNav />` and a `<main class="mx-auto max-w-5xl px-6 py-8"><RouterView /></main>`. Remove the create-vue starter markup/`<style>`.
8. **`BrandMark.vue` + `AppNav.vue`.** BrandMark: an inline `.pollen-dot bloom` span beside `<span class="font-display ...">Pollendar</span>`, wrapped in a `RouterLink` to `/`. AppNav: sticky top bar `border-b border-line bg-surface/70 backdrop-blur` with BrandMark left and an empty right-side `<slot name="actions" />` (auth phase fills it). Keep it small вАФ no data fetching here.
9. **API client `src/lib/api/client.ts`.** Export `const API_BASE = import.meta.env.VITE_API_BASE ?? "/api"`. Export `apiFetch<T>(path, init?)` that calls `fetch(\`${API_BASE}${path}\`, { credentials: "include", headers: { "Content-Type": "application/json", ...init?.headers }, ...init })`, parses JSON (tolerating empty 204 bodies), and on `!res.ok` throws `new ApiError(res.status, body)`. Export an `ApiError extends Error` class with `status: number` and `body: unknown` so callers can branch on `err.status === 409` (duplicate email/slot), `401` (no session), `400` (bad slotId). Add thin helpers `get/post/patch/del` if convenient вАФ but do **not** add any auth-token handling (the JWT lives in an httpOnly cookie the SPA never sees).
10. **API types `src/lib/api/types.ts`.** Define exactly the wire shapes from the backend, all ids `string`:
    - `type PollStatus = "open" | "completed" | "cancelled"` and `type Availability = "available" | "maybe" | "unavailable"`.
    - `interface User { id: string; email: string; displayName: string | null }` (matches `GET /api/auth/me` / `POST /api/auth/verify` `{ user }`).
    - `interface PollSlot { id: string; startTime: string | null; endTime: string | null; isAllDay: boolean; label: string | null; sortOrder: number }`.
    - `interface PollDate { id: string; eventDate: string; sortOrder: number; slots: PollSlot[] }`.
    - `interface PublicPoll { id; title; description: string | null; timezone: string; status: PollStatus; dates: PollDate[] }` (from `GET /api/public/polls/:token`).
    - `interface PollSummary { id; publicToken; shareUrl; title; status: PollStatus }` (from `POST /api/polls`) and a fuller `Poll` with nested dates+slots (from `GET /api/polls/:id`).
    - `interface SlotTally { slotId: string; available: number; maybe: number; unavailable: number; score: number }` and `interface BestSlot { slotId: string; date: string; label: string | null; score: number }`, `interface PollResults { best: BestSlot | null; slots: SlotTally[] }` (from `GET /api/public/polls/:token/results`).
    - `interface SubmitResponsesDto { displayName: string; email?: string; answers: { pollSlotId: string; availability: Availability }[] }` and its response `{ publicToken: string }` (the **participant** token вАФ distinct from the poll's URL token).
11. **`src/lib/utils/timezone.ts`.** Export `formatDate(eventDate, timeZone)` (e.g. "Thu Jun 26"), `formatTime(hms, timeZone)` parsing `"HH:mm"`/`"HH:mm:ss"`, `formatSlotRange(slot, timeZone)` collapsing start/end or "All day", and `localZoneLabel()` returning `Intl.DateTimeFormat().resolvedOptions().timeZone`. Use native `Intl.DateTimeFormat` with the `timeZone` option only вАФ no new dependency.
12. **UI primitives.** Build the five components from the mockup classes:
    - `Button.vue` вАФ primary: `inline-flex items-center justify-center gap-2 rounded-xl bg-pollen px-4 py-2.5 font-medium text-canvas shadow-glow transition hover:brightness-110 active:translate-y-px`; secondary: `border border-line bg-surface вА¶ hover:bg-surface2 text-moonlight`; ghost: `text-dim hover:text-moonlight` (no border/fill); danger: `border border-coral/40 bg-coral/10 text-coral hover:bg-coral/20`. Disabled вЖТ `cursor-not-allowed bg-pollen/40 text-canvas/60` (primary). `loading` вЖТ `aria-busy` + spinner SVG, button disabled. (Mockup: `mockups/components/buttons.html`.)
    - `Input.vue` вАФ `w-full rounded-xl border border-line bg-canvas px-4 py-3 text-moonlight placeholder:text-mute focus:border-pollen focus:outline-none focus:ring-2 focus:ring-pollen/30`; `invalid` вЖТ `border-coral focus:ring-coral/30` + `aria-invalid`; supports `type` (text/email) and a `textarea` variant. (Mockup: `mockups/components/form-fields.html`.)
    - `Field.vue` вАФ `<label class="mb-1.5 block text-sm font-medium text-moonlight">` + default slot for the control + `hint` (`text-xs text-mute`) and `error` (`text-xs text-coral` with вЪ†) rendered conditionally; passes `invalid` down via slot props or wraps the control.
    - `Card.vue` вАФ `rounded-2xl border border-line bg-surface p-6 shadow-card sm:p-8`, optional `title` slot in `font-display text-2xl font-semibold tracking-tight`.
    - `Pill.vue` вАФ `inline-flex items-center rounded-full px-3 py-1 text-xs`; tones: neutral (`bg-surface2 text-dim`), pollen (`bg-pollen/15 text-pollen ring-1 ring-pollen/40`), mint (`bg-mint/15 text-mint`), coral (`bg-coral/15 text-coral`). Used for "gathering responses" / "completed" / "cancelled" / "Copied".
13. **Routes placeholder.** Trim the generated `src/router/index.ts` to a single `/` route rendering a minimal `Landing.vue` placeholder (real landing built in the auth phase) so the shell renders and typecheck/build pass. Remove `HomeView`/`AboutView`. Keep `createWebHistory(import.meta.env.BASE_URL)`.
